로그인 흐름

MASTER 계정 → OTP 방식으로 검증 (DB 조회 없음, 환경변수 기반) -> 최초 배포 시 설정 꼭 해줘야함
일반 계정 → DB에서 해시/솔트로 비밀번호 검증
세션이 만료된 상태라면 → 이전 화면 복원
최초 등록 관리자 계정 + 비밀번호 변경 이력 없음 → 강제 비밀번호 변경 후 재로그인
로그인 성공 → 어드민 모드 체크 여부에 따라 화면 분기 (User 화면 or Patient 화면)

화면 진입 분기

MASTER 계정 → 무조건 관리자(User) 화면
관리자 계정 + 어드민 모드 체크 → 관리자(User) 화면
일반 계정 or 어드민 모드 미체크 → 환자(Patient) 화면

계정 수정 정책

MASTER 계정 → ID + 비밀번호 모두 변경 가능 (모든 사용자 대상)
일반 관리자 계정 → 비밀번호만 변경 가능
최초 등록 관리자 계정 (USER_ID = 1) → 일반 관리자는 수정 불가, MASTER만 수정 가능 

계정 삭제 정책

최초 등록 관리자 계정 → MASTER 포함 누구도 삭제 불가 
관리자 최소 1명 유지 → 마지막 관리자는 해임 불가

어드민 체크박스 노출 조건

ID 입력 후 PasswordBox 포커스 진입 시 → DB에서 관리자 ID 목록 조회 후 해당 ID면 체크박스 노출

## 1. 로그인 인증 흐름 (Authentication)

### 🛡️ 계정 권한별 인증 방식
* **MASTER 계정**
    * **검증 방식:** OTP(One-Time Password) 기반
    * **특징:** DB 조회 없이 **환경변수(Environment Variable)** 기반으로 동작
    * **필수 사항:** 최초 배포 시 시스템 환경변수 설정 확인 필수 ( 수동 등록 or 레지스트 파일로 최초 설치 시 작업해야함 ) 
* **일반/관리자 계정**
    * **검증 방식:** DB 조회 기반 (Password Hash + Salt 적용)

### 🔄 로그인 후속 절차
1. **초기 비밀번호 강제 변경:** - 최초 등록된 관리자 계정 중 비밀번호 변경 이력이 없는 경우
   - 강제 변경 화면으로 이동 시킨 후 재로그인 유도
2. **화면 진입 분기:**
   - **관리자(User) 화면:** - MASTER 계정 로그인 시 (무조건)
     - 관리자 계정으로 로그인 + `어드민 모드` 체크박스 선택 시
   - **환자(Patient) 화면:** - 일반 계정 로그인 시
     - 관리자 계정이나 `어드민 모드` 미체크 시

## 2. 계정 관리 정책 

| 대상 | 수정 권한 | 삭제 권한 |
| :--- | :--- | :--- |
| **MASTER 계정** | **ID/PW 모두 변경 가능** (모든 사용자 대상) | 삭제 불가 (시스템 관리용) |
| **최초 관리자 (ID:1)** | 관리자 1명 정책 유지하면 자유롭게 수정 삭제 가능  | 관리자 최소 1명 유지 시 삭제 가능 |
| **일반 관리자** | 본인의 **비밀번호만 변경 가능** | 일반 관리자 간 삭제/수정 불가 |

## 3. UI/UX 설명

### 🔘 어드민 체크박스 노출 조건 (WPF)
* **트리거:** `ID` 입력 후 `PasswordBox`에 포커스 진입하는 시점
* **동작 프로세스:**
    1. 입력된 ID 값을 기반으로 DB 내 관리자 ID 목록 비동기 조회
    2. 조회 결과 해당 ID가 관리자 권한을 가진 경우에만 `Admin Mode` 체크박스 노출
* **목적:** 일반 사용자에게 불필요한 옵션을 노출하지 않기 위해
